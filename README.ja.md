# JPSupport

Lazarus の統合開発環境(IDE)のソースエディタで日本語入力を可能にするパッケージです。

## これは何ですか？

Linux版のLazarusでは、ソースコードを編集する画面に日本語を直接入力することができませんでした。

JPSupportをインストールすることで、この問題が解決され、ソースエディタに日本語コメントや文字列を自然に入力できるようになります。

## 主な機能

- 日本語の変換中に、入力中の文字に下線が表示されます
- 変換候補ウィンドウがカーソルの近くに表示されます
- 複数のファイルをタブで開いていても、正しく動作します

## 動作環境

- OS：Linux（Debian、Ubuntuなど）
- Lazarus 2.2.6以降
- Free Pascal Compiler 3.2.2以降
- 画面描画システム(GTK2)バックエンド
- 日本語入力：Fcitx5またはIBus（Mozc推奨）
- 動作確認済み：Raspberry Pi 4（ARM64）/ Debian 12 / Fcitx5 + Mozc、IBus + Mozc
- Raspberry Pi 5・x86_64での動作は未確認です（情報提供歓迎）

## インストール方法

### 準備

まず、このリポジトリの `package` フォルダ内のファイルを、お使いのパソコンの適当な場所にコピーしてください。

### 手順

Lazarusには「パッケージ」という仕組みがあり、機能を追加できます。JPSupportもこの仕組みを使ってインストールします。2つのパッケージを順番にインストールする必要があります。

**1つ目のパッケージ（JPSupport）：**

1. Lazarusを起動します
2. 上部メニューの「パッケージ」→「パッケージファイルを開く(.lpk)」をクリック
3. ファイル選択画面で `JPSupport.lpk` を探して開きます
4. 「パッケージ」という小さなウィンドウが開きます
5. ウィンドウ上部の「コンパイル」ボタンをクリック
6. 続いて「インストール」ボタンをクリック
7. 「本当にインストールしますか？」と聞かれたら「はい」をクリック
8. Lazarusが自動的に再ビルドを始めます（数分かかります）

**2つ目のパッケージ（JPSupportIDE）：**

再ビルド後、Lazarusが再起動したら、同じ手順で今度は `JPSupportIDE.lpk` をインストールします。

### 確認

2つ目のインストール後にLazarusが再起動したら、ソースエディタをクリックして日本語を入力してみてください。変換候補ウィンドウが表示されれば成功です。

## 技術的な説明（開発者向け）

GTK2の`gtk_key_snooper_install`を使用してキーイベントをグローバルに横取りし、独自のGtkIMContextに転送します。標準のLazarusビルドでは`synedit.ppu`が`Gtk2IME`フラグなしでコンパイルされているため、通常の方法では日本語入力ができません。JPSupportはLazarusの内部を改変せずにこの問題を解決しています。

主要ファイル：
- `JPSupportAdapter.pas` — コアとなるアダプタクラス
- `JPSupportIDEMain.pas` — Lazarusへの組み込み処理
- `JPSupportUnit.pas` — スタンドアロン版コンポーネント

## 既知の制限事項

- 画面描画システム(GTK2)バックエンドのみ対応（GTK3・Qt版は未対応）
